プロフィール生成ツール

Google Sheets のデータを読み込み、トレーディングカード（PPTX）と詳細プロフィールスライド（PPTX）を自動生成する Next.js 製ツール。このドキュメントを読めば同等のシステムを再構築できる。

ツール概要

スプレッドシートを編集せずに、人物紹介資料をワンクリックで生成する。AI（GPT-4o）が紹介文を自動生成し、pptxgenjs がブラウザ内で PPTX を組み立てて直接ダウンロードする。

生成できる成果物

トレーディングカード
名前・キャッチフレーズ・スキル・AI紹介文。5種類のカラーテーマ対応。PPTX 形式で出力。
詳細プロフィール（16:9）
左パネル＋右エリアの2カラム構成。代表案件・価値の出し方・こだわりを1枚のスライドに凝縮。PPTX 形式で出力。

API Route: /api/read-sheet Server

spreadsheetId のみ送ると全シート名を返す（Step 1）。sheetName も送るとそのシートのキー/値マップを返す（Step 2）。2段階にすることでドロップダウン選択を実現。

app/api/read-sheet/route.ts TypeScript 
import { google } from "googleapis";

// 認証インスタンスはモジュールスコープで1度だけ生成（毎リクエストで復号しない）
const auth = (() => {
  const b64 = process.env.GOOGLE_SERVICE_ACCOUNT_CREDENTIALS_B64;
  if (!b64) throw new Error("GOOGLE_SERVICE_ACCOUNT_CREDENTIALS_B64 is not set");
  const credentials = JSON.parse(Buffer.from(b64, "base64").toString("utf-8"));
  return new google.auth.GoogleAuth({
    credentials, scopes: ["https://www.googleapis.com/auth/spreadsheets.readonly"],
  });
})();

const SPREADSHEET_ID_RE = /^[a-zA-Z0-9_-]{20,60}$/;

export async function POST(request: NextRequest) {
  const { spreadsheetId, sheetName } = await request.json();

  // spreadsheetId のフォーマット検証
  if (!spreadsheetId || !SPREADSHEET_ID_RE.test(spreadsheetId))
    return Response.json({ error: "Invalid spreadsheet ID" }, { status: 400 });

  const sheets = google.sheets({ version: "v4", auth });
  const meta = await sheets.spreadsheets.get({ spreadsheetId });
  const sheetNames = meta.data.sheets
    ?.map((s) => s.properties?.title ?? "").filter(Boolean) ?? [];

  if (!sheetName) return Response.json({ sheetNames, data: {} });

  // sheetName を取得済み一覧で検証（レンジインジェクション防止）
  if (!sheetNames.includes(sheetName))
    return Response.json({ error: "Invalid sheet name" }, { status: 400 });

  const res = await sheets.spreadsheets.values.get({
    spreadsheetId, range: `'${sheetName}'!A:B`,
  });
  const data: Record<string, string> = {};
  for (const row of res.data.values ?? []) {
    const key = (row[0] ?? "").trim();
    const value = (row[1] ?? "").trim();
    if (key && !key.startsWith("##")) data[key] = value;
  }
  return Response.json({ sheetNames, data });
}

API Route: /api/summarize-card Server

スプレッドシートデータを受け取り、GPT-4o で 80〜100 字の営業用紹介文を生成する。氏名を主語にしない、体言止め禁止など、商用シーンに適した制約をプロンプトに組み込む。

app/api/summarize-card/route.ts TypeScript 
import OpenAI from "openai";
const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

function cap(s: string, n = 500) { return s.slice(0, n); }

export async function POST(request: NextRequest) {
  const data = ((await request.json()).data) as Record<string, string>;

  // 各フィールドを上限付きで取得（大量入力によるトークン浪費・インジェクション緩和）
  const name         = cap(data["氏名"] ?? "", 50);
  const engineerType = cap(data["エンジニアタイプ"] ?? "");
  const catchphrase  = cap(data["キャッチフレーズ"] ?? "");
  const reason       = cap(data["エンジニアをやっている理由・原動力"] ?? "");
  const episode      = cap(data["自慢のエピソード"] ?? "");

  // system/user ロール分離でプロンプトインジェクションを緩和
  // データは user ロールの JSON として渡す
  const response = await client.chat.completions.create({
    model: "gpt-4o",
    messages: [
      {
        role: "system",
        content: [
          "紹介文を生成するコピーライターです。",
          `- 「${name}は〜」のような氏名主語の書き出しは禁止`,
          "- 体言止めは使わず、自然な文体で",
          "- 80〜100文字に厳密に収めること",
          "- 紹介文のみを出力（見出し・説明不要）",
        ].join("\n"),
      },
      { role: "user", content: JSON.stringify({ name, engineerType, catchphrase, reason, episode }) },
    ],
    max_tokens: 120,  // 80〜100字 ≒ 30〜40トークン。余裕を持たせつつ過剰生成を防ぐ
  });
  const summary = response.choices[0]?.message?.content?.trim() ?? "";
  return Response.json({ summary });
}

fitText — テキスト切り詰めヘルパー

pptxgenjs の shrinkText / fit:"shrink" は OOXML の <a:normAutofit/> に依存するが、実際の動作が不安定。JS 側で文字数を計算して切り詰める方が確実。

app/lib/generatePptx.ts — fitText() TypeScript 
function fitText(
  text: string, wIn: number, hIn: number,
  fontSize: number, lineSpacing = 1.3, isBold = false
): string {
  // 1文字幅: pt → inch 換算。日本語/太字は 0.65、通常は 0.58
  const charW = (fontSize * (isBold ? 0.65 : 0.58)) / 72;
  // 1行高さ: pptxgenjs 内部余白補正として 1.15 を掛ける
  const lineH = (fontSize / 72) * lineSpacing * 1.15;
  const charsPerLine = Math.max(1, Math.floor(wIn / charW));
  const maxLines     = Math.max(1, Math.floor(hIn / lineH));
  const maxChars     = charsPerLine * maxLines;
  if (text.length <= maxChars) return text;
  return text.slice(0, maxChars - 1) + "…";
}

案件カードの詳細テキスト配置

⚠️ 重要な落とし穴：pptxgenjs のテキストボックスはデフォルトで anchor="ctr"（垂直中央揃え）。複数テキストボックスに分割すると、背の高いボックス内でテキストが中央に配置され余白に見える。役割・規模・追加フィールドを1つのテキストボックスに結合することで解決。
app/lib/generatePptx.ts — 案件カード詳細部分 TypeScript 
const titleY  = py + 0.08;
const detailY = titleY + TITLE_H + 0.04;
const detailH = projH - (detailY - py) - RES_H - 0.04;

// meta（役割・期間・規模）と extra フィールドを1ボックスに統合
// → anchor:ctr による上余白問題を構造的に解決
const detailLines = [
  ...(meta ? [meta] : []),
  ...extras.map(({ label, value }) => `${label}: ${value}`),
];
if (detailLines.length > 0) {
  slide.addText(fitText(detailLines.join("\n"), projW-0.84, detailH, 10, 1.4), {
    x: RX+0.84, y: detailY, w: projW-0.84, h: detailH,
    fontSize: 10, color: C.slate, fontFace: "Meiryo UI",
    wrap: true, lineSpacingMultiple: 1.4,
    valign: "top", margin: 0,  // margin:0 = 内部余白ゼロ（lIns/tIns/bIns/rIns すべて）
  });
}

PPTX 生成: トレーディングカード Client

generateCardPptx(fields, themeId)。16:9 スライドの中央にカードを配置する。テーマ色（5種）は EngineerCard.tsx と同じ値をミラーリング。

app/lib/generatePptx.ts — generateCardPptx() 主要部分 TypeScript 
export function generateCardPptx(
  fields: { name: string; engineerType: string; catchphrase: string;
            summary: string; skills: string },
  themeId: string = "white"
) {
  const CW = 3.2;   // カード幅 (in)
  const CH = 5.0;   // カード高さ (in)
  const CX = (SW - CW) / 2;  // 水平中央配置
  const CY = (SH - CH) / 2;  // 垂直中央配置
  const AVH = 1.1;  // アバターエリア高さ

  // アバター角丸の下半分を塗りつぶして直角に見せるテクニック
  slide.addShape(pptx.ShapeType.rect, {
    x: CX, y: CY, w: CW, h: AVH, fill: { color: theme.avatarBg },
    line: { color: theme.border, width: 0 }, rectRadius: R,
  });
  slide.addShape(pptx.ShapeType.rect, {  // 下半分を角丸なしで上書き
    x: CX, y: CY + AVH * 0.5, w: CW, h: AVH * 0.5,
    fill: { color: theme.avatarBg }, line: { color: "FFFFFF", width: 0 },
  });

  // テキストボックスは可変の BY を使って積み上げ配置
  let BY = CY + AVH + 0.16;
  // engineerType → catchphrase → divider → summary → skills
  // 各要素の高さを推定して BY を加算していく

  pptx.writeFile({ fileName: `${fields.name || "card"}_card.pptx` });
}

// 全 API 呼び出しを集約。x-internal-token ヘッダーを必ず付与する
async function apiFetch(path: string, body: unknown) {
  return fetch(path, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",

    body: JSON.stringify(body),

}

実装済みのセキュリティ対策

以下の対策を実装済み。新規プロジェクトに流用する際も同様の構成を推奨する。

ミドルウェア認証（middleware.ts）

全 /api/* リクエストに対してトークン検証を行う。ルートごとに認証を書く必要がなく、追加した API Route に自動適用される。

middleware.ts TypeScript 
import { NextRequest, NextResponse } from "next/server";

export function middleware(req: NextRequest) {
  const token = req.headers.get("x-internal-token");
  if (token !== process.env.INTERNAL_API_TOKEN)
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  return NextResponse.next();
}

export const config = { matcher: ["/api/:path*"] };
⚠️ NEXT_PUBLIC_INTERNAL_API_TOKEN はクライアントバンドルに含まれブラウザから確認できる。URLを知っている人であれば誰でも利用可能な点は変わらない。ネットワークレベルの制御（VPN・Cloudflare Access 等）と組み合わせることで完全に遮断できる。

入力検証

spreadsheetId のフォーマット検証
Google Sheets ID は /^[a-zA-Z0-9_-]{20,60}$/ で検証。不正な文字列で Google API を呼び出す前に弾く。
sheetName のホワイトリスト検証
シート名は spreadsheets.get で取得した一覧との完全一致で検証する。'foo'!A1:ZZZ のようなレンジインジェクションを防止。
プロンプトへの入力上限
各フィールドを最大 500 字で切り詰める cap() 関数を適用。GPT-4o への過剰トークン送信とインジェクション文字列の規模を制限。

エラーハンドリング

API Route の catch ブロックはエラーをサーバーログにのみ出力し、クライアントには汎用メッセージを返す。Google API / OpenAI SDK のエラーメッセージにはサービスアカウント情報等が含まれる場合があるため、そのまま返却しない。

エラーハンドリングパターン TypeScript 
} catch (err) {
  console.error("[route-name]", err);  // サーバーログのみ
  return Response.json({ error: "処理に失敗しました" }, { status: 500 });
}

pptxgenjs の落とし穴

テキストボックスのデフォルト垂直揃えは中央（anchor = "ctr"）
pptxgenjs ソース line 2581: anchor = TEXT_VALIGN.ctr。テキストボックスを大きくすると内部でテキストが中央に配置され、上に余白ができる。valign: "top" を明示することで anchor="t" に上書きされる。それでも余白が出る場合は複数ボックスを1つに統合して根本解決する。
margin vs inset — 内部余白の指定
inset: 0 は lIns / tIns / bIns をゼロにするが rIns は変わらない。margin: 0 を使うと4辺すべてが valToPts(0) でゼロになる。Y 座標をテキスト先頭に正確に合わせたい場合は margin: 0 が正解。
shrinkText / fit:"shrink" は信頼できない
どちらも OOXML の <a:normAutofit/> を挿入するが、PowerPoint/LibreOffice のレンダリング依存で動作が不安定。JS 側で fitText() を実装して文字数を計算し、超過分を … に切り詰めるほうが確実。
日本語と英数字が混在するとフォントが混在する
pptxgenjs はデフォルトで Latin 文字に Calibri、CJK に別フォントを適用する。バッジの「案件 1」のように混在するとサイズ感がずれる。fontFace: "Meiryo UI" を明示することで統一される。
ブラウザキャッシュで generatePptx が古いままになる
dynamic import() でロードされるモジュールは初回アクセス後にブラウザにキャッシュされる。コードを変更しても反映されない場合は Ctrl+Shift+R（ハードリロード）で解消する。開発中は next dev のホットリロードが効かないケースがある。
rectRadius は pptxgenjs の独自拡張（インチ単位）
OOXML の custGeom を内部で生成する。単位はインチ。0.05 程度が視覚的に自然な角丸になる。
